## LoRA 合并与评估操作指南

### 1️⃣ 合并 LoRA 权重到基座模型

```bash
# 合并 4 层 LoRA
python LoRA/merge_lora.py --layers 4

# 合并 8 层 LoRA
python LoRA/merge_lora.py --layers 8

# 合并 16 层 LoRA
python LoRA/merge_lora.py --layers 16
```

合并后输出：`out/merge_{layers}_768.pth`

### 2️⃣ 评估模型

**方式 A：合并后模型直接评估**
```bash
python LoRA/eval.py
```
会自动评测：`full_sft`(基座)、`lora_4`(LoRA)、`merge_4`(合并后)

**方式 B：实时加载 LoRA 推理对话**
```bash
# 使用 LoRA 权重推理
python eval_llm.py --lora_weight lora_4 --lora_top_layers 4

# 使用合并后权重推理
python eval_llm.py --weight merge_4
```

### 核心代码逻辑

**合并公式**：
```python
# W_new = W_original + B @ A
lora_weight = torch.matmul(module.lora.B.weight, module.lora.A.weight)
module.weight.data = original + lora_weight
```

**评估方式对比**：
| 方式          | 优点                 | 缺点                    |
| ------------- | -------------------- | ----------------------- |
| LoRA 实时加载 | 节省显存，可切换权重 | 推理时多一步计算        |
| 合并后权重    | 推理更快更简单       | 合并后无法单独保留 LoRA |

你有现有的 LoRA 权重文件在 `out/lora/` 目录下，直接运行合并命令即可。
